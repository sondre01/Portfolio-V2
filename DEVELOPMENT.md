# Portfolio V2 - Development Journey

## Overview
Portfolio V2 is a complete redesign and rebuild of my personal portfolio, created to showcase both my **Software/Web Development** and **IT Systems/DevOps Administration** capabilities. This iteration prioritizes modern aesthetics, responsive design, and a professional tech stack that reflects my current skill set.

---

## 🎯 Vision & Goals

### What I Wanted to Achieve:
1. **Dual Expertise Showcase**: Create a single platform that effectively presents both web development and IT/DevOps knowledge
2. **Premium Aesthetics**: Move beyond basic portfolio templates to create a truly modern, visually appealing experience
3. **Responsive & Accessible**: Ensure the portfolio works seamlessly across all devices and meets accessibility standards
4. **Functional Backend**: Implement working features (contact forms, email delivery) rather than static placeholders
5. **Production-Ready**: Build with deployment in mind using modern cloud infrastructure

---

## 🛠️ Technology Stack & Why I Chose Each

### Frontend
- **HTML5**: Semantic markup for better SEO and accessibility
- **CSS3 (Vanilla)**: No build tools needed; modular stylesheets organized by section for maintainability
  - Glassmorphic designs and custom scrollbars for premium feel
  - Responsive grid layouts using CSS Grid and Flexbox
  - Dark mode toggle with localStorage persistence
- **JavaScript (Vanilla ES6)**: Lightweight client-side interactivity without framework overhead
  - IntersectionObserver for scroll-triggered animations
  - AJAX for asynchronous form submission
  - Dynamic modal rendering for project details

### Backend & Infrastructure
- **Vercel Serverless Functions**: Scalable, cost-effective backend for email handling
- **Node.js (ES6)**: Server-side JavaScript for API logic
- **Resend API**: Reliable email delivery service with templating capabilities
- **Live Server**: Local development environment on port 5501

### Design & Assets
- **Poppins + Inter Fonts**: Premium typography pairing
- **Lucide Icons**: Modern, consistent icon system
- **Devicon & Brandfetch CDNs**: Dynamic tech logo delivery
- **Custom SVG Decorations**: Unique visual elements for branding

## 💡 Technical Highlights

### Why Vanilla JavaScript?
- No build process needed
- Faster load times
- Greater control over functionality
- Perfect for a portfolio showcasing technical fundamentals

### Why Modular CSS?
- Easy to maintain and scale
- Clear organization (one file per section)
- Faster to debug styling issues
- Demonstrates CSS mastery

### Why Serverless Functions?
- Zero server management
- Automatic scaling
- Pay only for usage
- Perfect for low-traffic contact forms

### Why Resend for Email?
- Reliable delivery
- Built for developers
- Simple API integration
- Professional email handling
